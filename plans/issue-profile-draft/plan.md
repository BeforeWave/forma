# Issue Plan

## Goal

- Add `forma profile draft` as the first adoption-core workflow: a local command that turns explicit project-rule source files into a reviewable Forma profile draft package for a human or agent to refine before tracked profile promotion.

## Plan Strategy

- Plan Strategy: step-execution

## Scope

- In scope:
  - Add a `profile` CLI group with `forma profile draft`.
  - Add deterministic local source collection and conservative rule extraction from explicit `--source` files or directories.
  - Write a draft package containing `profile.draft.yaml`, `missing-decisions.md`, and `agent-review.md`.
  - Require `--profile-id <kebab>` and make the generated `profile.draft.yaml` pass the existing `load_profile()` gate.
  - Document the draft workflow in English and Chinese command/profile-authoring docs.
- Out of scope:
  - Calling an external LLM, network service, or agent from the CLI.
  - Automatic whole-project scanning without explicit `--source`.
  - Generating `resources`, `conditional_overlays`, or source adapters automatically.
  - Changing `create-bundle`, `create-plugin`, `install`, `verify`, `explain`, methodology, creator source, committed generated examples, or `dist/` release artifacts.

## Approach

- Add `src/forma/profile_draft.py` for source discovery, line/block classification, draft YAML rendering, report rendering, output overwrite handling, and `load_profile()` self-check.
- Add `@main.group("profile")` and `profile draft` in `src/forma/cli.py`.
- CLI contract:
  - Required: `--profile-id <kebab>`, at least one `--source <file-or-dir>`, and `--output <dir>`.
  - Optional: `--bundle-name <kebab>` defaulting to `--profile-id`, `--org-name <name>` defaulting to `Local Team`, and `--replace`.
  - Directory sources read only `.md`, `.txt`, `.yaml`, and `.yml`.
  - Directory traversal skips `.git/`, `dist/`, `examples/generated/`, `.forma-workflow/`, and `__pycache__/`.
  - Existing output directories are rejected unless `--replace` is set.
- Extraction contract:
  - High-confidence durable rules become concise `constraints.<stage>` entries.
  - High-confidence validation commands become `validation_commands.<stage>` entries.
  - Ambiguous, heavy, route-specific, private, adapter-like, or one-off material goes to `missing-decisions.md`, not into YAML.
  - `constraints.default` stays minimal; broad reading, generated-baseline, governance, and route-specific requirements are never placed there by default.
  - `profile.draft.yaml` is a candidate artifact, not source of truth until reviewed and moved into a tracked profile location by the user.

## Artifact/Evidence Boundary

- Source-of-truth refs: current conversation decisions take precedence; the Forma grounding handoff supplies repository facts and validation surfaces.
- Committed source artifacts:
  - New or edited source under `src/forma/`.
  - New tests under `tests/`.
  - English and Chinese docs under `docs/`.
  - This issue's plan, task, and run evidence under `plans/issue-profile-draft/`.
- Transient validation artifacts:
  - Smoke-test source and generated outputs under `/private/tmp/forma-profile-draft-smoke/`.
  - Draft packages generated by the command are local user artifacts and are not committed by default.
- Forbidden generated/release paths:
  - No edits to `dist/`, `examples/generated/`, `source/methodology/`, `source/skill-creator/`, or `profiles/forma-self/`.
- Evidence paths:
  - Task evidence is recorded under `plans/issue-profile-draft/runs/`.
  - `agent-review.md` inside a generated draft package records source paths, skipped paths, extraction summary, missing decisions, and self-check result for that local package.
- State policy:
  - Preserve unrelated worktree changes.
  - Do not install generated draft outputs.
  - Do not treat generated draft packages as durable profile source without later user review.

## Constraints

- Keep the first version deterministic and reproducible; no model calls, credentials, or network assumptions.
- Keep parsing conservative and explainable; false negatives belong in `missing-decisions.md`, false positives should be avoided in YAML.
- Keep public examples sanitized; do not copy private paths, credentials, or long source excerpts into generated docs or examples.
- Preserve existing CLI command behavior and help semantics except for adding the new profile draft route.

## Acceptance Criteria

- `forma profile draft --profile-id <kebab> --source <file-or-dir> --output <dir>` writes exactly the draft package files `profile.draft.yaml`, `missing-decisions.md`, and `agent-review.md`.
- Generated `profile.draft.yaml` passes `load_profile()` and can be used as input to `forma create-bundle`.
- The command rejects missing sources, unsupported source types, invalid profile or bundle ids, and existing output directories without `--replace`.
- Directory sources include only supported document-rule file types and skip the settled forbidden directories.
- High-confidence rules and validation commands are classified into stage-specific YAML entries; uncertain material is visible in `missing-decisions.md`.
- CLI help and docs make clear that profile draft is reviewable candidate output, not an automatic promotion to durable tracked profile source.

## Risks / Notes

- Conservative extraction may miss useful rules; this is acceptable for v1 because the generated package is meant to seed agent/human review rather than replace it.
- If a source document mixes durable policy with one-off instructions, the extractor must prefer `missing-decisions.md` over overfitting the profile YAML.
