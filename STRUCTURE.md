# Repository Structure

This document describes the **intended** Forma source tree. The repository is currently in bootstrap; most paths below are planned and not yet present.

## Top level

| Path | Role | Status |
|---|---|---|
| `README.md` | Human-facing project overview | present |
| `AGENTS.md` | Agent-facing repo entrypoint | present |
| `STRUCTURE.md` | This file — top-level structure map | present |
| `LICENSE` | MIT license text | present |
| `plans/issue-<id>/` | Per-issue planning and execution state | present per issue |
| `source/` | Methodology source layers (plan-first base + overlays + business-injection points) | planned |
| `tools/` | Layer 2 verifier and Layer 3 generator implementation | planned |
| `examples/` | Reference skill suites generated by Forma | planned |
| `tests/` | Test suite for the verifier and the generator | planned |

## Planning state per issue

| Path | Role |
|---|---|
| `plans/issue-<id>/plan.md` | Goal, Scope, Approach, Validation, Plan Strategy, Constraints, Acceptance Criteria |
| `plans/issue-<id>/tasks.md` | Structured task checklist (`Accept:` / `Validate:` / optional `Use-Check:` / `Depends:` / `Constraint:`) |
| `plans/issue-<id>/runs/` | Per-task execution evidence |
| `plans/issue-<id>/implement_notes.md` | Execution decision journal (optional, added when decisions matter to later tasks or review) |

## Notes

- Path names under "planned" rows are intentional but tentative; later issues may revise them as the methodology source and tooling take shape.
- This file should be updated whenever a planned path is materialized or renamed.
